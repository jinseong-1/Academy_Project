## 📕amend(어멘드)
 -> 커밋할때 기록남기지 않는 것</br>
소스트리에서 마지막 커밋 정정</br>
    -> 이렇게하면 소스트리에 기록이 안남음(수정은 가능)</br>
커밋 메세지 변경도 똑같이하면 에러가 발생하는데 이 하나만 차이나면 소스트리에서 강제푸시 시키면 됨

## 📗reset
- 이 커밋까지 현재 브랜치 초기화</br>
<br/>
🤓종류<br/>
    1. ☝Mexed : 변경사하아은 남아있지만 커밋만 날리는 것.
    2. ✌soft : 커밋메세지만 날리고 스테이징은 그대로 남아있음
    3. 👌Hard : 커밋내용, 파일까지 전부 날리는 것<br/>

Revert : reset은 없애지만, 이전으로 돌리고 그 작업내용이 남아있음<br/>
Stash(스테시) : 쉽게 말하면 임시저장소
 여러 브랜치로 왔다갔다하다가 다시 돌아올땐 소스트리에서 왼쪽하단에 stash에서 되돌릴수 있음<br/>

 ### 원격 저장소에서 되돌리기
 Pull requests -> closed -> 찾고자하는 파일 -> 승인 메시지 옆에 revert가 있음

# 📖 GitBush 명령어
___
- pwd : 현재 폴더 확인
- ls : 현재 폴더의 파일
- ls -a : 숨긴파일까지 전부 표시
- cd : 홈 폴더로 이동(ex. c:/User/사용자명)
- cd ~ : ~로 이동
- mkdir<새폴더 이름> : 새로운 폴더 생성
- git init -~ : git 폴더를 생성하고 ~브랜치를 생성 가능
- git config --list : 현재 프로젝트의 모든 옵션 확인
- git status : 위킹트리의 상태 보기
- git init -b main : git 저장소 생성 + main 브랜치 초기화
- git config --global <옵션명> : 전역 옵션 확인
- git config --global <옵션명> "<새로운값>" : 전역 옵션 <b>설정</b>
- git config --global --unset<옵션명> : 전역 옵션 삭제
- git config --local <옵션명> : 지역 옵션 확인
- git config --local <옵션명> "<새로운값>" : 지역 옵션 설정
- git config --local --unset<옵션명> : 지역 옵션 삭제
- ctrl+c : 잘못된 명령어를 입력하여 ">"가 나왔을때 탈출하는 커멘드

<b>지역 옵션이 전역 옵션을 우선한다(local을 통해서 프로젝트 별 다른 옵션 설정 가능)</b>

## 깃 색상 추가

## GitBush 에서 폴더 및 파일 생성
- echo "~" : "~" 라고 메아리가 생김
- echo "~" > : "~" 파일명 생성
- echo "~~" >> : 파일에 내용 추가
- git reset ~ : ~라는 스테이징한 파일을 다시 내리는 것
- 관례 : vs code로 commit할때 첫째줄은 제목, 둘째줄은 비우고 셋째줄이 본문인데 너무 길면 보기 좋지않아 50자 이내로 줄바꿈해주기

## git log 
### 자세한 기록을 보고 싶을 때(HEAD관련된 커밋만)
- git log --oneline : 간단한 커밋해시와 제목만 보고 싶을 때
- git log --oneline --graph --all --decorate : 모든 브랜치를 보고싶을 때
- git log --oneline -n : 보고싶은 최신 커밋의 갯수
